# タスク定義書 (anaslo-scraper-ja)

本タスク定義書は、要件定義および設計に基づき、開発・検証に必要な具体的な作業タスクを定義します。

---

## 1. 実装タスク一覧

### 基準表記
*   `[ ]` 未着手
*   `[/]` 進行中
*   `[x]` 完了

---

### Phase 1: 環境構築と設定 (目標: 開発準備完了)
*   `[x]` **TASK-1: 開発ディレクトリの準備と依存ライブラリの定義** (tools/anaslo-scraper/配下の作成)
*   `[x]` **TASK-2: 設定ファイルの作成** (config.jsonの作成と店舗設定)

### Phase 2: データベースとクローラーの実装 (目標: データ蓄積の自動化)
*   `[x]` **TASK-3: データベースヘルパー (`db.py`) の実装**
    *   SQLiteの初期化（3つのテーブルとインデックスの自動作成）処理を書く。接続時のタイムアウト設定（`timeout=20.0`）を付与。
    *   特定の日付のデータがすでに登録済みか確認する関数を実装する。
    *   パースしたデータをトランザクションを用いて一括保存する関数を実装する。
    *   リフレッシュ用のデータ削除関数（`delete_daily_data`）を実装する。
    *   起動時の自動バックアップローテーション（最大5世代管理、`backup/` ディレクトリ配下）処理を実装する。
*   `[/]` **TASK-3-EXT: データベースの拡張と移行**
    *   `unit_details` テーブルに `bb_probability`, `rb_probability` カラムを追加。
    *   既存DBに対する `ALTER TABLE` 移行ロジックの実装。
*   `[x]` **TASK-4: アナスロパーサー・クローラー (`scraper.py`) の実装**
    *   店舗一覧ページから日付リンクの一覧を取得する処理を実装する。
    *   個別ページから「全体データ（総差枚、勝ち台数など）」と「台別詳細テーブル」をBeautifulSoupでパースする処理を実装する（パチンコデータは除外）。
    *   パースしたデータを `db.py` 経由でデータベースに保存する結合ロジックを実装する。
    *   通常モード（未登録収集）、バックフィルモード（指定日数遡り収集）、リフレッシュモード（指定日上書き）の各収集ロジックを実装する。
    *   実行間のスリープ（3〜5秒）、User-Agentの設定、リトライ処理（最大3回、10秒間隔、15秒タイムアウト）、およびエラー時の安全な終了処理を実装する。
    *   複数店舗の実行で、1店舗がエラーになっても後続店舗の収集を続行する例外ハンドリングを追加する。
*   `[/]` **TASK-4-EXT: パーサー・クローラーの改善**
    *   `BB確率` `RB確率` のパース対応。
    *   店舗一覧のページネーション（複数ページ）対応。

### Phase 3: ダッシュボードとランナーの実装 (目標: 分析機能の完成)
*   `[x]` **TASK-5: Streamlitダッシュボード (`app.py`) の実装**
    *   店舗選択、期間（30/60/90/カスタム）選択、機種名絞り込み用のサイドバーUIを作成する。
    *   店舗全体のKPIカード（平均差枚、勝率、勝ち台数/総台数、稼働）を実装する。
    *   「曜日別」「日付の末尾別」「台番号の末尾別」の集計SQLと棒グラフ（AltairまたはStreamlit標準）を実装する。勝率は「勝ち台数÷総台数」で算出。
    *   「7のつく日」「ゾロ目の日」などのイベント日と通常日の平均差枚数比較棒グラフを実装する。
    *   期間内の強機種ランキングテーブルを実装する。
*   `[/]` **TASK-5-EXT: ダッシュボードの修正**
    *   イベント日判定の修正（5の日→7の日）。
    *   `BB確率` `RB確率` の表示追加。
*   `[x]` **TASK-6: 総合起動スクリプト (`run.py`) の実装**
    *   引数（`--collect`, `--backfill`, `--refresh`, `--dashboard`）に応じた自動実行ロジックを実装する。
    *   引数がない場合に表示されるCLIの対話メニューを実装する。
    *   `logs/` ディレクトリ自動作成と、ローテーション（10MB上限）対応のログ出力（`logs/anaslo.log`）を設定する。
*   `[/]` **TASK-6-EXT: ランナーの修正**
    *   ダッシュボード起動時のバックアップ処理追加。

### Phase 4: 検証と仕上げ (目標: システムの引き渡し)
*   `[ ]` **TASK-7: 動作検証と受け入れテストの実施**
    *   `scraper.py` のバックフィルモードを動かし、テスト店舗（大盛空港通り店）の過去データが正しく蓄積されるか確認する。
    *   二重登録防止が正しく機能するか、また上書き更新でデータが正しくリフレッシュされるか検証する。
    *   ネットワーク切断やconfig.json破損などの異常系エラーが安全にハンドリングされ、ログに記録されるか検証する。
    *   `app.py` を動かしてダッシュボード上でデータが正しく集計・表示されるか確認する。
*   `[ ]` **TASK-8: 簡易マニュアル（README）の作成**
    *   Windows PCでシステムを動かすための「Pythonのインストール方法」「起動コマンド」「タスクスケジューラでの自動化手順（引数付き起動設定）」をまとめたREADMEを作成する。
